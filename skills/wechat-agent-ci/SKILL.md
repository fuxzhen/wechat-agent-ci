# 🧙‍♂️ Skill: 微信小程序全栈开发助手 (V3.0)

## 📋 能力简介

一句话需求，自动完成微信小程序开发全流程：
- 需求分析 → 代码生成 → 预览测试 → 部署发布

适用场景：任何微信小程序功能开发，不限行业/类型

---

## 🛑 核心铁律：SOP 执行顺序

当收到用户的一句话需求时，**严格禁止**立刻写代码！必须按 4 个阶段执行：

### 阶段 1：需求评审与可视化确认
输出【方案确认书】，包含：
1. **代码逻辑过程**：用自然语言说明修改哪些文件、用什么 API、数据流向
2. **UI 界面可视化**：Markdown 表格或 ASCII 展示修改前后对比

**暂停等待用户确认后才能进入下一阶段**

### 阶段 2：测试驱动开发 (TDD)
收到"确认"后，先在 `__tests__` 目录编写测试用例

### 阶段 3：执行编码与自测修复
修改代码满足测试，执行自测直到 100% 通过

### 阶段 4：预览与发版
生成二维码给用户确认，确认后执行部署

---

## 📖 示例：如何输出【方案确认书】

用户需求："帮我在首页加个签到按钮，点击能领积分"

回复：
> 老板，收到需求。请评审以下方案：

> 
> **确认后开始开发？**

## ⚙️ 项目配置

### 初始化项目
1. 复制 `.env.example` 为 `.env`，填入 AppID 和密钥路径
2. 放置 `private.key` 到项目根目录

### 目录结构（默认）
```
miniprogram-3/
├── preview.js      # 生成预览二维码
├── deploy.js      # 上传发布
├── pages/         # 页面代码
├── cloudfunctions/ # 云函数
└── components/   # 组件
```

---

## 🚀 常用命令

```bash
# 生成预览二维码
node preview.js

# 上传发布
node deploy.js

# 语法检查
node -c pages/xxx.js
```

## ⚠️ 禁止事项

- ❌ 严禁直接在 main 分支修改代码
- ❌ 严禁未生成预览二维码就提交
- ❌ 严禁未通过自测就发布
- ❌ 严禁未经用户确认就开发
